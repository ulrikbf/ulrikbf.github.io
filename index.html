1 Je suis un blog

Since i became interested in EMACS, I've read countless blogs and articles about everything from how to configure the most minute detail about EMACS, to how to install and configure large packages. I've grown quite fond of reading well written blogs, and thought to myself it would be a good exercise and a challenge to write one myself! So without further ado, here it is, my journey with Emacs!

1.1 My blogging tools

I wandered fare and wide, searching for something that was fast, easy to use and compatible with org-mode. I first looked at static-site generators, like Jekyll and org-page. Org-page seemed to best suite my need, with a simple interface, layout and fully compatible with org-mode. Org-page also had the benefit of using git as version control, and separated the html/css/js in one branch while keeping the .org files on master. It sounded really good in theory, but i just could not get it to work in my environment(Windows). After doing some more research, i landed with hexo.

1.1.1 Hexo

Going from zero to hero, or from a simple file to a fully fledged blog was so incredibly simple with Hexo, i almost could not believe it! The recipe was as follows:

  • Download Hexo and Node.js
  • Navigate to Hexo/Blog
  • Download hexorenderer-org
  • Create a .org file in Hexo/Blog/sources/post
  • Run hexo generate to create a the HTML and hexo server to start up a node.js instance of your blog on localhost.
  • That's it!

Hexo even comes with full git integration, just defined your repo in _config.yml and run hexo generate -d. After tinkering around with the config file, I'm really glad i didn't get Org-page to work, even though that seemed to be the perfect fit! Changing the look and feel of your blog is also god damn simple, just clone a theme from hexo-themes and add the name of your theme in the hexo/blog/config.yml, and you're good to go! The themes are really different from each other, and I'm certain you would find a theme that would match your taste. Now this really turned into quite the sale pitch for Hexo, and i can't say that i mind.

1.1.2 hexorender-org

Hexo supports several 'backends' or rendering-tools. It support tools for markdown, swig, jade and even org. I found hexorenderer-org to fufill my needs. Now all i need is an .org file, and org-html-export and the rendering engine of hexo-render-org.

1.1.3 (provide 'Hexo)

Well, i just couldn't help it, i had to create a package for Hexo. It's not complete yet, but i will soon make it available on github. Now i only need to invoke(in emacs) hx/create-post and hx/deploy. Those two commands are what made this entry and pushed it to github.

2 To summary, all you need it a little bit of love and:

  • Node.js
  • Emacs
